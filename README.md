## 🎯 Краткое описание

**Hypothesis Engine** — это локальный веб-инструмент, который помогает исследователям и инженерам быстро генерировать научные гипотезы для решения технологических задач. Вы указываете целевой KPI (например, «повысить жаропрочность сплава на 15%»), загружаете литературу (статьи, патенты, отчёты, изображения, Excel) или просто вводите текстовый вопрос, а система за несколько минут выдаёт список проверяемых гипотез с обоснованием, ссылками на источники, оценкой новизны, рисков и влияния на KPI. Инструмент работает локально (данные не передаются в облако) и поддерживает три LLM-провайдера: локальный Ollama, бесплатный G4F и Yandex GPT.

**Ключевые преимущества:**
- 🚀 **Скорость:** формулировка гипотез — от недель до минут.
- 🧠 **Прозрачность:** каждая гипотеза содержит обоснование, механизм, источники и объяснение.
- 🌍 **Мультиязычность:** интерфейс и генерация на русском, английском, китайском.
- 🔍 **Поиск в интернете:** автоматический сбор источников из DuckDuckGo (без ключей).
- 📚 **RAG-база знаний:** векторный поиск по загруженным документам.
- 📊 **Визуализация:** интерактивный граф знаний, дорожная карта проверки.
- 📤 **Экспорт:** JSON, TXT, PDF (бизнес-отчёты).

## ✨ Основные возможности

- **Загрузка данных:** текст, PDF (включая сканы с OCR), изображения (распознавание текста), Excel/CSV.
- **Извлечение сущностей:** автоматическое выделение материалов, свойств, параметров, связей (регулярные выражения).
- **Граф знаний:** построение и визуализация (NetworkX + pyvis).
- **Генерация гипотез:** использование LLM с полями: *statement, mechanism, novelty, risk, impact, sources, explanation, recommendation*.
- **Ранжирование:** многокритериальная оценка с настраиваемыми весами (новизна, влияние, риск).
- **Поиск источников:** DuckDuckGo (без API-ключей, поддержка разных языков).
- **RAG-модуль:** загрузка документов в локальную базу знаний, векторный поиск (Sentence-Transformers + FAISS).
- **Дорожная карта:** редактируемый план проверки для каждой гипотезы.
- **Обратная связь:** лайк/дизлайк, корректировка весов на основе фидбэка.
- **Метаданные:** авторы, дата публикации, источник.
- **Мультиязычность:** русский, английский, китайский (интерфейс и генерация).
- **Экспорт:** JSON, TXT, PDF-отчёты (reportlab).

## 🏗 Архитектура решения

```
[Входные данные]
    │
    ├── KPI + ограничения (ручной ввод)
    ├── Литература (текст, PDF, изображения, Excel/CSV)
    ├── Простой текстовый вопрос (без загрузки)
    │
    ▼
[Модуль загрузки и OCR]
    │
    ├── PyPDF2 / pdf2image / Tesseract / EasyOCR
    ├── pandas (Excel/CSV)
    │
    ▼
[NLP-извлечение (EntityExtractor)]
    │
    ├── Материалы, свойства, параметры (регулярные выражения)
    ├── Отношения (повышает, снижает, образует и др.)
    │
    ▼
[Граф знаний (NetworkX)]
    │
    ├── Визуализация (pyvis)
    ├── Текстовое представление для LLM
    │
    ▼
[Обогащение контекста]
    │
    ├── Поиск в интернете (DuckDuckGo)
    ├── RAG-база знаний (Sentence-Transformers + FAISS)
    │
    ▼
[Генератор гипотез (HypothesisDeliberation)]
    │
    ├── Провайдеры: Ollama (локально), G4F (бесплатно), Yandex GPT
    ├── Промпт с учётом языка (ru/en/zh)
    ├── Парсинг JSON-ответа
    │
    ▼
[Ранжирование (Scorer)]
    │
    ├── Веса: новизна, влияние, риск (настраиваемые)
    ├── Сортировка по убыванию score
    │
    ▼
[Интерфейс (Streamlit)]
    │
    ├── Отображение графа, гипотез, источников, дорожной карты
    ├── Обратная связь, настройка весов, выбор провайдера
    │
    ▼
[Экспорт]
    ├── JSON
    ├── TXT
    └── PDF (reportlab)
```

## 💻 Требования к системе

### Минимальные требования

- **ОС:** Windows 10/11, macOS 11+, Linux (Ubuntu 20.04+)
- **Python:** 3.12 или выше (рекомендуется 3.12)
- **Оперативная память:** 8 ГБ (для локальных моделей Ollama – 16 ГБ)
- **Место на диске:** минимум 5 ГБ (для моделей и библиотек)
- **Интернет:** для G4F и поиска в интернете (опционально)

| **Git** | Система контроля версий | [git-scm.com](https://git-scm.com/downloads) |
| **Ollama** (опционально) | Локальный сервер для LLM | [ollama.com](https://ollama.com/download) |

## 📦 Установка и настройка

### Шаг 1. Клонирование репозитория

```bash
git clone https://github.com/Anfihal/hypothesis-engine.git
cd hypothesis-engine
```

### Шаг 2. Создание и активация виртуального окружения

**Windows (PowerShell):**
```bash
python -m venv venv
.\venv\Scripts\Activate.ps1
```

**macOS/Linux:**
```bash
python3 -m venv venv
source venv/bin/activate
```

Если вы используете несколько версий Python, укажите конкретную:
```bash
python3.12 -m venv venv
```

### Шаг 3. Установка Python-зависимостей

Создайте файл `requirements.txt` (или используйте приложенный) со следующим содержимым:

```txt
# Core
streamlit>=1.28.0
networkx>=3.0
pyvis>=0.3.2

# LLM и интеграция
langchain>=0.1.0
langchain-community>=0.0.10
g4f>=7.7.1

# PDF, OCR, изображения
PyPDF2>=3.0.0
pdf2image>=1.16.0
pytesseract>=0.3.10
Pillow>=10.0.0
easyocr>=1.7.0
numpy>=1.24.0

# Данные
pandas>=2.0.0
openpyxl>=3.1.0

# RAG
sentence-transformers>=5.6.0
faiss-cpu>=1.14.3
langchain-text-splitters>=1.1.2
pypdf>=6.14.2

# Экспорт
reportlab>=4.0.0

# Поиск
duckduckgo-search>=6.0.0

# Утилиты
python-dotenv>=1.0.0
requests>=2.31.0
```

Установите их:

```bash
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Если у вас мало места на диске, используйте флаг `--no-cache-dir`:
```bash
python -m pip install --no-cache-dir -r requirements.txt
```

### Шаг 4. Установка внешних компонентов (опционально)

#### 4.1. Ollama (для локальных моделей)

**Windows:** скачайте установщик с [ollama.com](https://ollama.com/download) и запустите.
**macOS:** `brew install ollama` или скачайте установщик.
**Linux:** `curl -fsSL https://ollama.com/install.sh | sh`.

Загрузите рекомендуемую модель (займёт ~5 ГБ):
```bash
ollama pull llama3.1:8b   # или llama3.2:3b (легче)
```

Запустите сервер (обычно запускается автоматически, но можно проверить):
```bash
ollama serve
```

#### 4.2. Tesseract OCR (для распознавания текста на изображениях)

**Windows:** скачайте установщик с [GitHub](https://github.com/UB-Mannheim/tesseract/wiki) и установите. Добавьте путь к `tesseract.exe` в системную переменную PATH (обычно `C:\Program Files\Tesseract-OCR`).
**macOS:** `brew install tesseract`
**Linux:** `sudo apt install tesseract-ocr`

#### 4.3. Poppler (для конвертации PDF в изображения)

**Windows:** скачайте архив [poppler-windows](https://github.com/oschwartz10612/poppler-windows/releases), распакуйте в `C:\poppler` и добавьте `C:\poppler\bin` в PATH.
**macOS:** `brew install poppler`
**Linux:** `sudo apt install poppler-utils`

Если вы не планируете использовать OCR, эти компоненты не обязательны.

### Шаг 5. Настройка конфигурации Streamlit (увеличение лимита загрузки)

Создайте папку `.streamlit` в корне проекта и файл `config.toml`:

```bash
mkdir .streamlit
echo "[server]
maxUploadSize = 4096" > .streamlit/config.toml
```

Это разрешит загрузку файлов до 4 ГБ.

### Шаг 6. Настройка Yandex GPT (опционально)

Если вы хотите использовать Yandex GPT, создайте файл `.env` в корне проекта:

```bash
echo "YANDEX_API_KEY=ваш_ключ" > .env
echo "YANDEX_FOLDER_ID=ваш_folder_id" >> .env
```

**Где взять ключи?** Зарегистрируйтесь в [Yandex Cloud](https://console.cloud.yandex.ru/), создайте сервисный аккаунт с ролью `ai.languageModels.user` и сгенерируйте API-ключ.

## 🖥️ Использование

### 1. Входные параметры

- **KPI** – опишите целевое свойство или технологическую проблему (например, «повысить жаропрочность сплава на 15%»).
- **Ограничения** – укажите доступные ресурсы, бюджет, оборудование (например, «без использования рения, бюджет < $500k»).
- **Литература** – введите текст вручную или загрузите файлы (txt, pdf, png, jpg, xlsx, csv). Можно оставить поле пустым – система сгенерирует гипотезы на основе только KPI и ограничений.

### 2. Настройка модели

В боковой панели выберите:
- **Провайдер:** `ollama` (локально), `g4f` (бесплатно, без ключей), `yandex` (требует ключи).
- **Модель:** для Ollama – `llama3.1:8b`, `llama3.2:3b`, `mistral:7b`; для G4F – `gpt-4o-mini`, `gpt-4o`, `claude-3-haiku` и др.
- **Включить поиск в интернете:** система найдёт дополнительные источники через DuckDuckGo.
- **Настройка весов ранжирования:** измените важность новизны, влияния и риска.

### 3. Генерация

Нажмите кнопку **«Сгенерировать гипотезы»**. Процесс займёт от 15 секунд до нескольких минут (зависит от выбранной модели и длины контекста).

### 4. Результаты

- **Граф знаний** – интерактивная визуализация извлечённых сущностей и связей.
- **Найденные источники** – список ссылок из интернета (если поиск включён).
- **Гипотезы** – каждая содержит:
  - заявление, механизм, новизну, риск, влияние,
  - список источников (с гиперссылками),
  - подробное объяснение,
  - рекомендацию по проверке.
- **Дорожная карта** – редактируемый план экспериментов (вы можете добавлять/удалять/изменять шаги).
- **Обратная связь** – кнопки «Полезно» / «Неполезно» для каждой гипотезы.

### 5. Экспорт

- **JSON** – полные данные с дорожной картой.
- **TXT** – текстовый отчёт.
- **PDF** – бизнес-отчёт (если установлен reportlab).

## 🐛 Устранение неполадок

| Проблема | Возможное решение |
|----------|-------------------|
| **Ошибка импорта модуля** | Установите недостающий пакет: `pip install <package>` |
| **Таймаут при генерации** | Увеличьте таймаут в `deliberation.py` (переменная `timeout`) или переключитесь на G4F |
| **Медленная работа Ollama** | Используйте модель `llama3.2:3b`, включите быстрый режим (галочка в интерфейсе) |
| **Ошибка 429 Yandex** | Превышен лимит запросов – переключитесь на G4F или Ollama |
| **Не отображаются ссылки на источники** | Убедитесь, что включён поиск в интернете (чекбокс) |
| **Не удаётся загрузить большой файл** | Проверьте настройку `maxUploadSize` в `.streamlit/config.toml` |
| **Ошибка OCR (Tesseract)** | Установите Tesseract и добавьте его в PATH; либо используйте EasyOCR (установлен по умолчанию) |
